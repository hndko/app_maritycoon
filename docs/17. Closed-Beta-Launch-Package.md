# Closed Beta Launch Package

Dokumen ini adalah paket operasional untuk mengundang 20-50 tester pertama MariTycoon. Rilis ini ditujukan untuk Closed Beta terbatas, bukan public production launch.

## 1. Closed Beta Launch Plan

### Tujuan

- Memvalidasi gameplay MVP dari room creation sampai winner.
- Menguji stabilitas realtime untuk room 2, 4, dan 8 pemain.
- Mengumpulkan feedback UX desktop dan mobile.
- Mengidentifikasi bug prioritas sebelum Open Beta.

### Scope Rilis

Included:

- Create room.
- Join via link dan room code.
- Public/private/password/invite-only room.
- Ready status dan host controls.
- Chat realtime dengan emoji.
- Dice, movement, property buy, rent, jail, bankruptcy, winner, Play Again.
- Reconnect dan disconnect handling.

Excluded:

- Login Google.
- Ranking.
- Achievement.
- Tournament.
- Voice chat.
- AI bot.
- Trading system.
- Spectator mode.
- Full auction.

### Launch Phases

| Phase | Target | Durasi | Gate |
| --- | --- | --- | --- |
| Internal staging validation | QA/dev team | 1-2 hari | Checklist 2/4/8 pemain PASS |
| Closed beta wave 1 | 10-15 tester | 1 sesi | Tidak ada Critical bug |
| Closed beta wave 2 | 20-30 tester | 1-2 sesi | Room 4P dan 8P stabil |
| Closed beta wave 3 | 50 tester max | 2-3 sesi | Bug High terkendali |

### Launch Gate

Closed Beta boleh dimulai jika:

- `docs/16. Staging-Validation-Checklist.md` critical flow PASS.
- `GET /api/health` status `ok`.
- HTTPS/WSS staging aktif.
- Backup berhasil dibuat.
- Restore drill staging berhasil minimal sekali.
- Load test 50 client memiliki hasil stabil.
- Rollback target commit dicatat.
- Moderator dan bug triage owner sudah ditentukan.

### Roles

| Role | Tanggung Jawab |
| --- | --- |
| Release Lead | Keputusan go/no-go, koordinasi tester, rollback decision |
| QA Lead | Menjalankan checklist, validasi bug, sign-off |
| Backend Owner | Memantau API, Socket.IO, Redis, PostgreSQL, logs |
| Frontend Owner | Memantau UX, console error, mobile/browser issue |
| DevOps/SRE | Memantau deployment, health, metrics, backup, rollback |
| Community/Moderator | Membantu tester, mengumpulkan feedback |

## 2. Tester Instructions

### Sebelum Mulai

1. Gunakan link staging yang diberikan moderator.
2. Gunakan browser terbaru: Chrome, Edge, Firefox, Safari, atau mobile browser.
3. Jangan gunakan data pribadi sensitif sebagai nickname.
4. Simpan room code saat menemukan bug.
5. Jika diminta menguji multi-player, gunakan incognito/profile browser berbeda.

### Yang Harus Dicoba

- Buat room sebagai host.
- Bagikan room code/link ke tester lain.
- Join room dari link.
- Join room dari room code.
- Uji invite-only room jika moderator meminta.
- Set ready dan mulai game.
- Roll dice saat giliran.
- Beli properti saat tersedia.
- Amati rent otomatis saat berhenti di properti pemain lain.
- Uji chat dan emoji.
- Refresh browser untuk menguji reconnect.
- Lanjutkan game sampai winner jika memungkinkan.
- Coba Play Again setelah game selesai.

### Yang Harus Dilaporkan

Laporkan bug jika:

- Tidak bisa create/join room.
- Room code/link salah.
- Player hilang dari room.
- Turn tidak berpindah.
- Dice, posisi, uang, atau property berbeda antar pemain.
- Rent tidak sesuai.
- Jail tidak bisa diselesaikan.
- Bankruptcy membuat game stuck.
- Winner tidak muncul.
- Play Again tidak reset.
- Browser console menampilkan error merah.
- Aplikasi lambat atau terputus.

### Informasi Wajib Saat Melapor Bug

- Nama tester.
- Device dan browser.
- Room code.
- Waktu kejadian.
- Langkah sebelum bug.
- Screenshot atau screen recording jika ada.
- Apakah bug bisa diulang.

## 5. Rollback Plan

### Trigger Rollback

Rollback dilakukan jika salah satu terjadi:

- Backend health degraded lebih dari 5 menit.
- Mayoritas room tidak bisa join atau start game.
- Banyak room stuck pada turn timer.
- Security issue pada private/password/invite-only room.
- Data room/game corrupt dan tidak bisa dipulihkan dengan restart.
- Bug Critical menghalangi mayoritas tester.

### Pre-Rollback Checklist

- Catat commit saat ini.
- Catat commit terakhir yang stabil.
- Buat backup jika database masih sehat:

```bash
./backup.sh
```

- Beri tahu tester bahwa sesi dihentikan sementara.

### Rollback Commands

Rollback ke commit stabil:

```bash
git fetch origin
git checkout <stable_commit>
./deploy.sh
```

Jika perlu restore data:

```bash
./restore.sh backups/<backup-before-incident>.dump
```

Verifikasi setelah rollback:

```bash
docker compose --env-file .env.production -f docker-compose.production.yml ps
curl https://your-domain.com/api/health
```

### Post-Rollback

- Catat root cause sementara.
- Simpan log backend dan screenshot error.
- Buat issue untuk bug Critical/High.
- Jangan lanjutkan Closed Beta sampai hotfix lolos smoke test.
