# Database Schema Design

Basis data menggunakan PostgreSQL untuk menyimpan data persisten seperti konfigurasi ruangan, state awal, dan profil sementara guest.
Untuk data yang sangat real-time (posisi pemain, state game aktif), akan di-handle in-memory oleh backend (Redis/Socket) namun di-sync secara periodik ke DB.

## Table: `users_guest`
Menyimpan data profil sementara pengunjung yang bermain.
- `id` (UUID, Primary Key)
- `nickname` (VARCHAR 50, Not Null)
- `created_at` (TIMESTAMP, Default NOW())

## Table: `rooms`
Menyimpan informasi server/room permainan yang dibuat.
- `id` (UUID, Primary Key)
- `room_code` (VARCHAR 10, Unique, Not Null) - *Contoh: ABCD1234*
- `room_name` (VARCHAR 100, Not Null)
- `password_hash` (VARCHAR 255, Nullable) - *Jika room private dengan password*
- `is_public` (BOOLEAN, Default TRUE)
- `max_players` (INT, Default 4)
- `starting_money` (INT, Default 15000000)
- `status` (ENUM: 'waiting', 'playing', 'finished')
- `created_by` (UUID, Foreign Key ke users_guest)
- `created_at` (TIMESTAMP, Default NOW())

## Table: `room_players`
Menyimpan data pemain di dalam suatu room.
- `id` (UUID, Primary Key)
- `room_id` (UUID, Foreign Key ke rooms)
- `user_id` (UUID, Foreign Key ke users_guest)
- `player_name` (VARCHAR 50)
- `money` (INT, Default 0)
- `position` (INT, Default 0) - *Indeks kotak di papan (0-39)*
- `is_bankrupt` (BOOLEAN, Default FALSE)
- `is_host` (BOOLEAN, Default FALSE)
- `turn_order` (INT) - *Urutan bermain*
- `joined_at` (TIMESTAMP)

## Table: `properties`
Data master (katalog) seluruh properti di papan monopoli versi Indonesia. Data ini statis.
- `id` (INT, Primary Key) - *Sesuai urutan di papan, misalnya 1-40*
- `name` (VARCHAR 100) - *Nama Kota, misal: Jakarta*
- `type` (ENUM: 'city', 'station', 'utility', 'tax', 'chance', 'community_chest', 'start', 'jail', 'parking')
- `color_group` (VARCHAR 20, Nullable) - *Contoh: 'Dark Blue', 'Red'*
- `price` (INT, Nullable) - *Harga beli tanah*
- `base_rent` (INT, Nullable) - *Harga sewa tanah kosong*
- `rent_1_house` (INT, Nullable)
- `rent_2_house` (INT, Nullable)
- `rent_3_house` (INT, Nullable)
- `rent_4_house` (INT, Nullable)
- `rent_hotel` (INT, Nullable)
- `house_price` (INT, Nullable) - *Harga beli 1 unit rumah/hotel*
- `mortgage_value` (INT, Nullable)

## Table: `room_properties`
Status kepemilikan dan pembangunan properti dalam suatu room spesifik.
- `id` (UUID, Primary Key)
- `room_id` (UUID, Foreign Key ke rooms)
- `property_id` (INT, Foreign Key ke properties)
- `owner_id` (UUID, Foreign Key ke room_players, Nullable)
- `house_count` (INT, Default 0)
- `hotel_count` (INT, Default 0)
- `is_mortgaged` (BOOLEAN, Default FALSE)

## Table: `game_logs`
Menyimpan histori kejadian (log) dari sebuah room untuk fitur Game Log dan Reconnect history.
- `id` (UUID, Primary Key)
- `room_id` (UUID, Foreign Key ke rooms)
- `event_type` (VARCHAR 50) - *Contoh: 'dice_roll', 'buy_property', 'pay_rent'*
- `payload` (JSONB) - *Detail event*
- `created_at` (TIMESTAMP)
